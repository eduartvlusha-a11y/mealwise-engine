# MealWise — Sale Checklist & Next Moves

This checklist defines the exact steps to sell MealWise
as a €100K+ engine-first asset with 0 users.

No feature additions are required.

---

## 1. Asset Freeze (MANDATORY)

Before outreach:
- No code changes
- No refactors
- No new features
- Only bug fixes if critical

Reason:
Buyers want stability, not velocity.

---

## 2. Sale Package (WHAT TO SEND)

The following files constitute the full package:
- docs/SYSTEM_OVERVIEW.md
- docs/ARCHITECTURE.md
- docs/EXTENSION_GUIDE.md
- docs/DEMO_WEEK.json
- docs/TRANSFER_PACKAGE.md
- docs/VALUATION_POSITIONING.md

Optional:
- Read-only repo access
- Short recorded walkthrough (10–15 min)

---

## 3. Buyer Pitch (ONE SENTENCE)

“MealWise is a deterministic weekly food decision engine
that replaces planning, grocery aggregation, budget control,
and substitution logic — transferable with zero users.”

Do not pitch features. Pitch leverage.

---

## 4. Buyer Qualification (IMPORTANT)

Target buyers who:
- build food-tech or retail systems
- already have users
- already have pricing or store data
- want to reduce R&D time

Avoid buyers asking about:
- App Store launch
- influencer marketing
- UI polish

They are not your buyers.

---

## 5. Outreach Channels

Primary:
- Direct outreach to food-tech founders / CTOs
- LinkedIn (technical buyers)
- Private introductions

Secondary:
- Flippa (strategic listing, high price anchor)
- MicroAcquire / Acquire.com (engine positioning)

---

## 6. Negotiation Rules (DO NOT BREAK)

- Never justify price emotionally
- Anchor with replacement cost (€150K–€300K)
- Emphasize transferability
- Emphasize risk reduction

If price is challenged:
“Building this internally costs more than acquiring it.”

---

## 7. Deal Structure (RECOMMENDED)

- Asset sale (code + docs)
- Optional 30–60 day transition support
- Optional license-back if you want future upside

Avoid revenue-share complexity.

---

## 8. Walk-Away Conditions

Walk away if buyer:
- asks for free custom features
- wants founder to stay long-term
- undervalues deterministic core

Scarcity increases price.

---

## 9. Post-Sale Options

After sale you may:
- reuse architecture for new markets
- build UI on top for B2C
- license a lighter version

MealWise becomes a platform, not a one-off.

---

## Status

When all items above are complete:
- Asset is READY
- €100K valuation is defensible
- Founder dependency is minimal
